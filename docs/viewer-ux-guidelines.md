# Viewer UX 设计说明

## 背景

本文档用于沉淀本次会话中已定型的 Viewer UX 方案，作为后续迭代、代码评审、跨端对齐和功能回归时的统一参考。

## 目标

- 让远程桌面画面始终成为主视觉主体。
- 让顶栏、底栏、面板层只在用户明确需要时出现。
- 避免应用内工具栏与 HarmonyOS 系统菜单、系统边缘操作发生冲突。
- 保持全屏与非全屏状态下的交互心智一致。
- 让 HarmonyOS 与 Android Viewer 尽量共享同一套交互语义，减少跨端学习成本。

## 核心原则

- 远程内容优先：鼠标移动、点击、拖拽、滚轮等常规输入优先服务远端桌面。
- 明确唤起：Viewer chrome 必须通过清晰、主动的动作唤起，而不是被边缘误触发。
- 心智稳定：系统级操作属于系统边缘，应用级操作属于应用把手和应用面板。
- 低打扰：用户回到远程画布后，工具栏应在短暂空闲后再次自动隐藏。

## 显隐模型

### 默认状态

- 未连接：
  - 展示会话准备区和必要的连接上下文。
  - 不隐藏关键连接操作。
- 已连接且首帧已就绪：
  - 顶栏和底栏在短暂延时后自动隐藏。
  - 面板层默认关闭。
- 已连接但首帧未就绪：
  - 保持必要状态提示可见，不提前进入沉浸隐藏。
- 面板已打开：
  - 固定显示顶栏/底栏。
  - 仅在面板关闭后恢复自动隐藏。
- 报错或连接中：
  - 优先保证状态信息可见。

### 自动隐藏规则

- 自动隐藏同时适用于全屏和非全屏的已连接查看状态。
- 仅在以下条件同时满足时启用自动隐藏：
  - 会话已连接；
  - 没有面板处于打开状态；
  - 没有编辑弹层处于打开状态；
  - 没有阻塞性的错误状态；
  - 已经拿到首帧画面。
- 用户通过明确入口打开 chrome 后，重新开始自动隐藏计时。
- chrome 显示期间隐藏右侧把手；chrome 收起后重新显示把手。

### 状态流转

- `Watch`：
  - 远程画布优先显示。
  - chrome 自动隐藏，把手作为唯一主入口存在。
- `Wake`：
  - 用户点击或轻触右侧把手。
  - chrome 显示，并开始新的自动隐藏计时。
- `Operate`：
  - 用户正在顶栏、底栏或面板中操作。
  - 自动隐藏暂停或延长，直到用户离开交互区域。
- `Back To Canvas`：
  - 面板关闭、工具栏空闲一段时间后，平滑回到远程画布优先状态。

## 唤起模型

### 定型方案

- 移除顶部和底部整条热区对应用 chrome 的唤起能力。
- 使用右侧中部把手作为 Viewer chrome 的主入口。

### 方案原因

- 顶部和底部边缘天然与 HarmonyOS 系统菜单、系统工具栏和系统手势区域重叠。
- 全宽边缘触发会在用户尝试系统操作时产生误唤起。
- 右侧中部把手可以把系统动作与应用动作分离，形成更清晰的用户心智模型。

### 把手行为

- 位置：
  - 放在右侧中部，必要时可略微下移以获得更平衡的视觉重心。
- 视觉：
  - 采用窄胶囊样式，空闲时低透明度显示。
  - 悬停或按下时可增强可见度。
- 行为：
  - 会话连接成功后显示。
  - 仅在会话已连接且首帧已经可见时显示，避免在空白加载阶段制造无效入口。
  - 点击或轻触把手即可显示 Viewer chrome。
  - chrome 展开时把手隐藏。
  - chrome 自动收起后把手重新出现。

## 交互规则

### 不应触发 chrome 显示

- 鼠标在画布上的普通移动
- 鼠标在画布上的普通点击
- 画布上的拖拽操作
- 画布上的滚轮操作
- 键盘输入或软键盘输入本身
- 顶部或底部系统边缘附近的悬停

### 应触发 chrome 显示

- 点击或轻触右侧中部把手
- 在 chrome 已显示时继续打开某个功能面板
- 像全屏切换这类必须给出本地 UI 反馈的显式本地操作

### chrome 交互保护

- 当用户正在工具栏区域内悬停时，暂停自动隐藏计时。
- 当用户点击、触摸或连续操作工具栏控件时，刷新并适当延长自动隐藏计时。
- 用户离开工具栏后，不立即收起，而是给出一段更宽松的缓冲时间，避免在连续操作中突然消失。

## 面板策略

- 低频操作放入面板，不再依赖长期悬浮按钮常驻在画布上。
- 面板采用底部抽屉式组织，承载以下内容：
  - 剪贴板；
  - 屏幕切换；
  - 更多操作；
  - 安全信息；
  - 开发者工具；
  - 会话面板。
- 面板打开期间：
  - 顶栏/底栏保持可见；
  - 自动隐藏暂停；
  - 用户关闭后平滑回到远程画布。

## 控制栏规范

### 信息架构

- Viewer chrome 由三层组成：
  - 顶栏：承载连接标题、安全状态、全屏和更多入口。
  - 底栏：承载当前会话中的高频即时操作。
  - 底部面板：承载低频、信息密集或需要更多空间的操作集合。
- 控制栏按钮必须按“高频主操作优先、低频扩展后置”的原则组织。
- 不允许把多屏、剪贴板、安全信息等低频功能直接塞回主底栏，避免重新变回工具堆叠。

### 顶栏规则

- 顶栏保持轻量，只展示：
  - 连接标题；
  - 简短状态副文案；
  - 安全状态信息胶囊；
  - 全屏切换；
  - `更多` 入口。
- 标题区优先保证单行可读，副文案只做弱提示，不承担操作说明。
- `更多` 是进入控制中心的稳定入口，位置固定在顶栏末尾。

### 底栏规则

- 底栏采用语义分组，必须实现为独立的“悬浮岛”（Floating Island）容器。
- 每组拥有独立的深色背景（如 `rgba(12,20,22,0.82)`）、大圆角（如 `borderRadius(22)`）及柔和阴影。
- 严禁使用占满屏幕全宽的单块大面板背景，必须通过内容自然撑开并限制最大宽度，以最大限度地让出远程桌面的可视区域。
- 当前定型分组为：
  - 第 1 组：`控制模式`、`查看模式`、`键盘`、`导航`、`更多`
  - 第 2 组：`缩小`、`倍率`、`放大`，多屏时追加 `屏幕`
- 默认目标是 2 排结构：
  - 第一排承担模式与输入切换；
  - 第二排承担缩放和显示范围相关操作。
- 不再继续扩展为 3 排常驻工具栏；如果信息增多，应优先进入底部面板，而不是向下加第三排。

### 按钮尺寸策略

- 控制栏按钮不使用完全等宽方案。严禁在代码中使用强制等分（如 Android 的 `weight(1f)` 或 HarmonyOS 的 `layoutWeight(1)`）。
- 必须采用“统一最小宽度 + 按内容自然扩展 + 拥挤时允许收缩”策略（如 HarmonyOS 中的 `constraintSize({ minWidth: 68 })` 配合 `flexShrink(1)`）：
  - 保持整体节奏整齐；
  - 避免短文案按钮过窄，保证足够的点击热区；
  - 避免长文案按钮被强行压缩后可读性下降；
  - 在小屏或多按钮拥挤时，允许安全地收缩以适应屏幕。
- 同组按钮之间保持一致高度、圆角、内边距和间距，优先保证一眼可扫描，而不是机械等分。

### 多屏入口规则

- 多屏相关操作不直接展开在主底栏中。
- 主底栏只保留一个 `屏幕` 入口，用于打开独立的显示范围面板。
- `屏幕` 面板中承载以下能力：
  - `全览`
  - `屏幕 1...N`
  - `编辑屏幕布局`
- 多屏面板属于低频但高价值能力，打开期间应暂停 chrome 自动隐藏。

### 视觉 Token

- Viewer chrome 必须使用统一的深色悬浮体系，避免 Android 和 HarmonyOS 各自漂移。
- 当前控制栏视觉 token 约定如下：
  - 主激活背景：`#2C4F54`
  - 主激活前景：`#FFFFFF`
  - 默认按钮背景：`rgba(255,255,255,0.08)`
  - 默认边框：`rgba(255,255,255,0.12)`
  - 分组面板背景：`rgba(12,20,22,0.78)`
- 激活态通过背景色强化，不依赖额外描边或高饱和点缀色。
- 面板、胶囊和按钮的色板必须共享同一 token 来源，避免同屏出现多套深色容器。

### 交互反馈

- 按钮点击、悬停、拖动导航键等本地操作都应刷新 chrome 的存活时间。
- 打开 `更多` 或 `屏幕` 面板后，chrome 进入显式操作态，不得在用户阅读或选择时自动隐藏。
- 关闭面板后重新开始自动隐藏计时，回到远程画布优先状态。

## 全屏与非全屏一致性

- 全屏和非全屏使用同一套唤起模型。
- 不再为不同窗口模式设计不同的边缘触发规则。
- 两种模式之间只允许存在视觉间距和系统栏处理差异，不改变用户召唤工具栏的方式。

## 跨端对齐要求

- HarmonyOS 与 Android 均遵循“画布优先、显式唤起、自动回落”的统一模型。
- 平台差异只体现在系统栏、安全区、窗口 inset 和基础组件实现方式上。
- 不允许某一端继续保留“点击画布即弹出工具栏”或“上下边缘热区唤起”这类旧交互。
- 文档中的 chrome 指顶栏、底栏和功能面板；把手不属于 chrome，而是 chrome 的独立入口。

## 实现约束

- 从鼠标和触摸事件中移除基于上下边缘位置的唤起逻辑。
- 保持画布层专注于远端输入透传。
- 在页面壳层中实现独立的右侧把手组件。
- chrome 显示状态只受以下因素驱动：
  - 连接状态；
  - 面板状态；
  - 把手的显式触发；
  - 自动隐藏计时器。

## 后续可扩展项

- 增加一次性的首用提示，告知用户通过右侧把手打开工具栏。
- 增加自动隐藏时间的可配置能力。
- 在不与远端输入冲突的前提下，再评估是否补充本地键盘快捷方式。
